## 项目概述

### 功能定位
FastGithub 是一个 Github 加速工具，解决以下问题：
- GitHub 网站打不开
- 用户头像无法加载
- Releases 无法上传下载
- git-clone、git-pull、git-push 失败

**注意**：该项目不具备"翻墙"功能，也没有相关计划。

### 部署方式
| 平台 | 部署方式 |
|------|----------|
| Windows 桌面 | 双击运行 FastGithub.UI.exe |
| Windows 服务 | `fastgithub.exe start/stop` |
| Linux 终端 | `sudo ./fastgithub` + 系统代理配置 |
| Linux 服务 | `sudo ./fastgithub start/stop` + 系统代理配置 |
| macOS | 双击运行 + 证书信任 + 系统代理配置 |
| Docker | docker-compose 一键部署 |

### 双模式实现原理

#### 模式一：Windows 数据包拦截模式

**核心机制**：使用 WinDivert 驱动进行网络层拦截

```
原始请求流程：
用户 -> DNS 查询 github.com -> 真实 DNS -> 真实 IP -> 连接失败

拦截后流程：
用户 -> DNS 查询 github.com 
    └─> WinDivert 拦截 UDP:53
        └─> 返回 127.0.0.1（本地回环）
            └─> 用户连接 127.0.0.1:443
                └─> WinDivert 拦截 TCP:443
                    └─> 重定向到本地 Kestrel 监听端口
                        └─> Kestrel 处理 -> Yarp 代理 -> 真实服务器
```

**关键代码位置**：
- `FastGithub.PacketIntercept/Dns/DnsInterceptor.cs` - DNS 拦截
- `FastGithub.PacketIntercept/Tcp/TcpInterceptor.cs` - TCP 端口重定向

#### 模式二：Linux/macOS HTTP 代理模式

**核心机制**：使用系统 HTTP 代理

```
配置流程：
1. FastGithub 监听端口 38457
2. 用户设置系统代理为 http://127.0.0.1:38457
3. 浏览器/系统 HTTP/HTTPS 请求发送到代理端口

请求流程：
用户请求 -> 系统代理 -> FastGithub:38457 
    └─> 解析域名 -> 智能 DNS -> 选择最优 IP
        └─> 建立连接 -> 转发请求
```

**关键代码位置**：
- `FastGithub.HttpServer/KestrelServerExtensions.cs:ListenHttpProxy()`
- `FastGithub.HttpServer/TcpMiddlewares/HttpProxyMiddleware.cs`

### 域名配置系统

FastGithub 支持灵活的域名配置，主要配置项：

```json
{

    "FallbackDns": [
```

**配置项说明**：

| 配置项 | 类型 | 说明 |
|--------|------|------|
| `TlsSni` | bool | 是否在 TLS 握手时发送 SNI |

| `TlsIgnoreNameMismatch` | bool | 是否忽略服务器证书域名不匹配 |
| `Timeout` | TimeSpan | 请求超时时长，如 "00:02:00" |
| `IPAddress` | IPAddress | 强制使用指定 IP |
| `Destination` | Uri | 请求目的地重定向 |
| `Response` | object | 直接响应（阻断请求），包含 StatusCode、ContentType、ContentValue |

## 核心模块详解

### 1. DNS 拦截模块（Windows）

#### 1.1 DNS 拦截原理

**位置**: `FastGithub.PacketIntercept/Dns/DnsInterceptor.cs`

**核心逻辑**：

```
DNS 拦截流程：
1. 使用 WinDivert 捕获所有 UDP:53 端口的数据包
2. 解析 DNS 请求报文
3. 检查请求域名是否匹配配置的域名列表
4. 如匹配，构造 DNS 响应，返回 127.0.0.1 或 ::1
5. 交换源/目标端口，发送伪造的响应
6. 刷新系统 DNS 缓存
```

**关键实现代码**：

```csharp
// 拦截过滤器：只捕获目标端口为 53 的 UDP 包
private static readonly Filter filter = Filter.True.And(f => f.Udp.DstPort == 53);

// 修改 DNS 数据包
unsafe private void ModifyDnsPacket(WinDivertPacket packet, WinDivertAddress addr)
{
    var result = packet.GetParseResult();
    var requestPayload = result.DataSpan.ToArray();

    // 解析 DNS 请求
    if (TryParseRequest(requestPayload, out var request) == false ||
        request.OperationCode != OperationCode.Query ||
        request.Questions.Count == 0)
    {
        return;
    }

    var question = request.Questions.First();
    if (question.Type != RecordType.A && question.Type != RecordType.AAAA)
    {
        return;
    }

    // 检查域名是否匹配配置
    var domain = question.Name;
    if (this.fastGithubConfig.IsMatch(question.Name.ToString()) == false)
    {
        return;
    }

    // 构造 DNS 响应，返回回环地址
    var response = Response.FromRequest(request);
    var loopback = question.Type == RecordType.A ? IPAddress.Loopback : IPAddress.IPv6Loopback;
    var record = new IPAddressResourceRecord(domain, loopback, this.ttl);
    response.AnswerRecords.Add(record);

    // 修改数据包并发送
    var writer = packet.GetWriter(packet.Length - result.DataLength);
    writer.Write(response.ToArray());
    
    packet.ReverseEndPoint();           // 交换源/目标
    packet.ApplyLengthToHeaders();      // 更新长度
    packet.CalcChecksums(addr);          // 重新计算校验和
    packet.CalcOutboundFlag(addr);
    
    addr.Flags |= WinDivertAddressFlag.Impostor;
}
```

### 2. TCP 端口拦截模块（Windows）

#### 2.1 端口重定向原理

**位置**: `FastGithub.PacketIntercept/Tcp/TcpInterceptor.cs`

**核心逻辑**：

```
端口重定向流程：
1. 捕获特定端口的 TCP 数据包
2. 入站流量：修改目标端口（如 443 -> 本地监听端口）
3. 出站流量：修改源端口（如 本地端口 -> 443）
4. 重新计算校验和
5. 发送修改后的数据包
```

**关键实现代码**：

```csharp
// 过滤器：捕获回环地址上的特定端口流量
this.filter = Filter.True
    .And(f => f.Network.Loopback)
    .And(f => f.Tcp.DstPort == oldServerPort || f.Tcp.SrcPort == newServerPort);

// 修改 TCP 端口
unsafe private void ModifyTcpPacket(WinDivertPacket packet, WinDivertAddress addr)
{
    var result = packet.GetParseResult();
    if (result.TcpHeader->DstPort == oldServerPort)
    {
        // 入站：目标端口 old -> new
        result.TcpHeader->DstPort = this.newServerPort;
    }
    else
    {
        // 出站：源端口 new -> old
        result.TcpHeader->SrcPort = oldServerPort;
    }
    addr.Flags |= WinDivertAddressFlag.Impostor;
    packet.CalcChecksums(addr);
}
```

#### 3.3 DNS 可用性检测

```csharp
private async ValueTask<bool> IsDnsAvailableAsync(IPEndPoint dns, CancellationToken cancellationToken)
{
    // 非 53 端口（如 dnscrypt-proxy）直接认为可用
    if (dns.Port != DNS_PORT)
    {
        return true;
    }

    // 检查缓存
    if (this.dnsStateCache.TryGetValue<bool>(dns, out var available))
    {
        return available;
    }

    // 尝试 TCP 连接检测（标准 DNS 应支持 TCP）
    using var socket = new Socket(dns.AddressFamily, SocketType.Stream, ProtocolType.Tcp);
    await socket.ConnectAsync(dns, linkedTokenSource.Token);
    return this.dnsStateCache.Set(dns, true, this.stateExpiration);
}
```

#### 3.4 IP 测速与排序

**位置**: `FastGithub.DomainResolve/DnsClient.cs:OrderByConnectAnyAsync`

```csharp
private static async Task<IList<IPAddress>> OrderByConnectAnyAsync(
    IList<IPAddress> addresses, int port, CancellationToken cancellationToken)
{
    // 并行连接所有 IP，选择第一个连接成功的
    var connectTasks = addresses.Select(address => ConnectAsync(address, port, linkedTokenSource.Token));
    var fastestAddress = await await Task.WhenAny(connectTasks);
    
    // 将最快的 IP 放在第一位
    if (fastestAddress == null || addresses.First().Equals(fastestAddress))
    {
        return addresses;
    }
    
    var list = new List<IPAddress> { fastestAddress };
    foreach (var address in addresses)
    {
        if (address.Equals(fastestAddress) == false)
        {
            list.Add(address);
        }
    }
    return list;
}
```

### 4. TLS 入侵与证书机制

#### 4.1 TLS 入侵检测

**位置**: `FastGithub.HttpServer/TlsMiddlewares/TlsInvadeMiddleware.cs`

**问题背景**：
- Kestrel 默认会对所有连接尝试 TLS 握手
- 对于非 TLS 流量（如 HTTP CONNECT 隧道），这会导致失败

**解决方案**：

```csharp
public async Task InvokeAsync(ConnectionDelegate next, ConnectionContext context)
{
    // 检测是否为 TLS 连接
    if (await IsTlsConnectionAsync(context) == false)
    {
        // 非 TLS 连接：注入假的 TLS 特性
        // 迫使后续的 TLS 中间件跳过处理
        if (context.Features.Get<ITlsConnectionFeature>() == null)
        {
            context.Features.Set<ITlsConnectionFeature>(FakeTlsConnectionFeature.Instance);
        }
    }
    await next(context);
}

// TLS 协议检测：检查前两个字节
// TLS 握手记录的第一个字节是 0x16 (Handshake)
// 第二个字节是主版本号 0x03 (TLS 1.x)
private static async Task<bool> IsTlsConnectionAsync(ConnectionContext context)
{
    var result = await context.Transport.Input.ReadAtLeastAsync(2, context.ConnectionClosed);
    var reader = new SequenceReader<byte>(result.Buffer);
    
    return reader.TryRead(out var firstByte) &&
           reader.TryRead(out var nextByte) &&
           firstByte == 0x16 &&
           nextByte == 0x3;
}
```

```csharp
private async ValueTask<Stream> ConnectCallback(
    SocketsHttpConnectionContext context, CancellationToken cancellationToken)
{
    var innerExceptions = new List<Exception>();
    
    // 1. 获取解析后的 IP 列表（已按测速排序）
    var ipEndPoints = this.GetIPEndPointsAsync(context.DnsEndPoint, cancellationToken);

    await foreach (var ipEndPoint in ipEndPoints)
    {
        try
        {
            // 2. 建立 Socket 连接
            var socket = new Socket(ipEndPoint.AddressFamily, SocketType.Stream, ProtocolType.Tcp);
            await socket.ConnectAsync(ipEndPoint, linkedTokenSource.Token);
            var stream = new NetworkStream(socket, ownsSocket: true);

            var requestContext = context.InitialRequestMessage.GetRequestContext();
            
            // 3. HTTP 请求：直接返回
            if (requestContext.IsHttps == false)
            {
                return stream;
            }

            // 4. HTTPS 请求：自定义 TLS 握手
            var tlsSniValue = requestContext.TlsSniValue.WithIPAddress(ipEndPoint.Address);
            var sslStream = new SslStream(stream, leaveInnerStreamOpen: false);
            
            await sslStream.AuthenticateAsClientAsync(new SslClientAuthenticationOptions
            {
                // 关键：自定义 SNI 值
                TargetHost = tlsSniValue.Value,
                // 关键：自定义证书验证
                RemoteCertificateValidationCallback = ValidateServerCertificate
            }, cancellationToken);

            return sslStream;
        }
        catch (Exception ex)
        {
            innerExceptions.Add(ex);
        }
    }
    
    throw new AggregateException("找不到任何可成功连接的IP", innerExceptions);
}
```

#### 5.4 SNI 控制机制

**位置**: `FastGithub.Configuration/TlsSniPattern.cs`

SNI（Server Name Indication）是 TLS 扩展，用于在握手时告诉服务器客户端要连接的域名。

FastGithub 支持以下 SNI 策略：

| 策略 | 配置值 | 效果 |
|------|--------|------|
| 发送真实域名 | `TlsSni: true` | SNI = 目标域名 |
| 不发送 SNI | `TlsSni: false` | SNI = null |
| 自定义模式 | `TlsSniPattern: "..."` | 按模板生成 |

**自定义模式变量**：
- `@domain` - 目标域名
- `@ipaddress` - 连接的 IP 地址
- `@random` - 随机值

**使用场景**：
- 某些 CDN 服务器根据 SNI 分流，不发送 SNI 可能获取更好的线路
- 某些被墙的域名，SNI 会被 GFW 检测

```csharp
public double GetRate()
{
    this.CleanInvalidRecords();  // 清理过期记录
    // 最近 5 秒的总字节数 / 5 = 速率
    return (double)this.queues.Sum(item => item.Length) / this.intervalSeconds;
}

private bool CleanInvalidRecords()
{
    if (Interlocked.CompareExchange(ref this.cleaning, 1, 0) != 0)
    {
        return false;  // 避免并发清理
    }

    var ticks = Environment.TickCount64;
    while (this.queues.TryPeek(out var item))
    {
        // 只保留最近 intervalSeconds 内的记录
        if (ticks - item.Ticks < this.intervalSeconds * 1000)
        {
            break;
        }
        else
        {
            this.queues.TryDequeue(out _);
        }
    }

    Interlocked.Exchange(ref this.cleaning, 0);
    return true;
}
```

```csharp
public class ConcurrentCache<TKey, TValue> where TKey : notnull
{
    private readonly ConcurrentDictionary<TKey, CacheEntry> _cache = new();
    private readonly TimeSpan _defaultTtl;
    private readonly Timer _cleanupTimer;
    
    private record CacheEntry(TValue Value, DateTimeOffset ExpiresAt);
    
    public TValue? GetOrAdd(TKey key, Func<TKey, TValue> factory, TimeSpan? ttl = null)
    {
        if (_cache.TryGetValue(key, out var entry) && entry.ExpiresAt > DateTimeOffset.Now)
        {
            return entry.Value;
        }
        
        var value = factory(key);
        var expiresAt = DateTimeOffset.Now.Add(ttl ?? _defaultTtl);
        _cache[key] = new CacheEntry(value, expiresAt);
        return value;
    }
}
```

### 7. 并发与异步优化

#### 7.1 当前问题

- `SemaphoreSlim` 用于每个 DNS 查询的并发控制
- 连接尝试是串行的（`await foreach`）
- 无超时的精细控制

#### 7.2 优化建议

**方案一：并行连接尝试**

```csharp
// 当前实现：串行尝试每个 IP
// await foreach (var ipEndPoint in ipEndPoints) { ... }

// 改进：有限并行
private async ValueTask<Stream> ConnectWithLimitedConcurrencyAsync(
    IAsyncEnumerable<IPEndPoint> ipEndPoints, 
    int maxConcurrency = 2,
    CancellationToken cancellationToken = default)
{
    using var semaphore = new SemaphoreSlim(maxConcurrency);
    using var cts = CancellationTokenSource.CreateLinkedTokenSource(cancellationToken);
    
    var tasks = new List<Task<(IPEndPoint EndPoint, Stream? Stream, Exception? Error)>>();
    
    await foreach (var endpoint in ipEndPoints)
    {
        await semaphore.WaitAsync(cts.Token);
        
        tasks.Add(Task.Run(async () =>
        {
            try
            {
                var stream = await ConnectSingleAsync(endpoint, cts.Token);
                cts.Cancel(); // 成功后取消其他
                return (endpoint, stream, null);
            }
            catch (Exception ex)
            {
                return (endpoint, null, ex);
            }
            finally
            {
                semaphore.Release();
            }
        }));
    }
    
    var results = await Task.WhenAll(tasks);
    var success = results.FirstOrDefault(r => r.Stream != null);
    
    if (success.Stream != null) return success.Stream;
    
    throw new AggregateException("所有连接尝试失败", 
        results.Select(r => r.Error).Where(e => e != null)!);
}
```

```rust
use tokio::net::TcpStream;
use tokio_rustls::{TlsConnector, TlsStream};

// 自定义证书验证器
struct SkipHostnameVerifier;

impl ServerCertVerifier for SkipHostnameVerifier {
    fn verify_server_cert(
        &self,
        _end_entity: &rustls::Certificate,
        _intermediates: &[rustls::Certificate],
        _server_name: &ServerName,
        _scts: &mut dyn Iterator<Item = &[u8]>,
        _ocsp_response: &[u8],
        _now: std::time::SystemTime,
    ) -> Result<ServerCertVerified, rustls::Error> {
        // 跳过域名验证（相当于 TlsIgnoreNameMismatch = true）
        Ok(ServerCertVerified::assertion())
    }
}

pub struct CustomTlsClient {
    connector: TlsConnector,
    sni_enabled: bool,
}

impl CustomTlsClient {
    pub fn new(sni_enabled: bool, ignore_hostname: bool) -> Self {
        let mut root_certs = RootCertStore::empty();
        
        // 加载系统根证书
        for cert in rustls_native_certs::load().unwrap() {
            root_certs.add(&rustls::Certificate(cert.0)).ok();
        }
        
        let mut config = ClientConfig::builder()
            .with_root_certificates(root_certs)
            .with_no_client_auth();
        
        // 自定义证书验证
        if ignore_hostname {
            config.dangerous()
                .set_certificate_verifier(Arc::new(SkipHostnameVerifier));
        }
        
        let connector = TlsConnector::from(Arc::new(config));
        
        Self {
            connector,
            sni_enabled,
        }
    }
    
    pub async fn connect(
        &self,
        addr: std::net::SocketAddr,
        domain: &str,
    ) -> Result<TlsStream<TcpStream>, String> {
        let tcp = TcpStream::connect(addr).await
            .map_err(|e| format!("TCP 连接失败: {}", e))?;
        
        // 控制 SNI
        let server_name = if self.sni_enabled {
            ServerName::try_from(domain.to_string())
                .map_err(|_| "无效的域名".to_string())?
        } else {
            // 不发送 SNI：使用 IP 地址或空值
            ServerName::IpAddress(addr.ip())
        };
        
        let tls = self.connector.connect(server_name, tcp).await
            .map_err(|e| format!("TLS 握手失败: {}", e))?;
        
        Ok(tls)
    }
}
```

```go
    if err != nil {
        return nil, fmt.Errorf("TCP 连接失败: %v", err)
    }
    
    // 配置 SNI
    tlsConfig := c.config.Clone()
    if c.sniEnabled {
        tlsConfig.ServerName = serverName
    } else {
        // 不发送 SNI：使用 IP 地址
        // Go 的 tls.Config 中 ServerName 为空则不发送 SNI
        tlsConfig.ServerName = ""
    }
    
    // TLS 握手
    tlsConn := tls.Client(tcpConn, tlsConfig)
    
    // 设置握手超时
    tlsConn.SetDeadline(time.Now().Add(10 * time.Second))
    if err := tlsConn.Handshake(); err != nil {
        tlsConn.Close()
        return nil, fmt.Errorf("TLS 握手失败: %v", err)
    }
    tlsConn.SetDeadline(time.Time{})
    
    return tlsConn, nil
}
```

**C 语言 TLS 客户端示例（OpenSSL）**：

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <arpa/inet.h>
#include <openssl/ssl.h>
#include <openssl/err.h>

typedef struct {
    SSL_CTX *ctx;
    int sni_enabled;
    int ignore_hostname;
} CustomTlsClient;

// 自定义证书验证回调
static int verify_callback(int preverify_ok, X509_STORE_CTX *ctx) {
    // 在这里可以实现自定义的验证逻辑
    // 返回 1 表示验证通过，0 表示失败
    return preverify_ok;
}

CustomTlsClient* tls_client_create(int sni_enabled, int ignore_hostname) {
    CustomTlsClient *client = malloc(sizeof(CustomTlsClient));
    if (!client) return NULL;
    
    // 初始化 OpenSSL
    SSL_load_error_strings();
    OpenSSL_add_ssl_algorithms();
    
    // 创建 SSL 上下文
    const SSL_METHOD *method = TLS_client_method();
    client->ctx = SSL_CTX_new(method);
    if (!client->ctx) {
        ERR_print_errors_fp(stderr);
        free(client);
        return NULL;
    }
    
    // 加载系统根证书
    SSL_CTX_set_default_verify_paths(client->ctx);
    
    // 设置验证模式
    if (ignore_hostname) {
        // 不验证主机名（但仍验证证书链）
        SSL_CTX_set_verify(client->ctx, SSL_VERIFY_PEER, verify_callback);
    } else {
        SSL_CTX_set_verify(client->ctx, SSL_VERIFY_PEER | SSL_VERIFY_FAIL_IF_NO_PEER_CERT, NULL);
    }
    
    client->sni_enabled = sni_enabled;
    client->ignore_hostname = ignore_hostname;
    
    return client;
}

SSL* tls_client_connect(CustomTlsClient *client, const char *ip, int port, const char *hostname) {
    // 创建 TCP socket
    int sockfd = socket(AF_INET, SOCK_STREAM, 0);
    if (sockfd < 0) {
        perror("socket");
        return NULL;
    }
    
    // 设置超时
    struct timeval tv = {.tv_sec = 10, .tv_usec = 0};
    setsockopt(sockfd, SOL_SOCKET, SO_RCVTIMEO, &tv, sizeof(tv));
    setsockopt(sockfd, SOL_SOCKET, SO_SNDTIMEO, &tv, sizeof(tv));
    
    // 连接服务器
    struct sockaddr_in serv_addr;
    serv_addr.sin_family = AF_INET;
    serv_addr.sin_port = htons(port);
    inet_pton(AF_INET, ip, &serv_addr.sin_addr);
    
    if (connect(sockfd, (struct sockaddr *)&serv_addr, sizeof(serv_addr)) < 0) {
        perror("connect");
        close(sockfd);
        return NULL;
    }
    
    // 创建 SSL 对象
    SSL *ssl = SSL_new(client->ctx);
    if (!ssl) {
        ERR_print_errors_fp(stderr);
        close(sockfd);
        return NULL;
    }
    
    // 设置 SNI（Server Name Indication）
    if (client->sni_enabled && hostname) {
        SSL_set_tlsext_host_name(ssl, hostname);
    }
    
    // 绑定 socket
    SSL_set_fd(ssl, sockfd);
    
    // TLS 握手
    if (SSL_connect(ssl) != 1) {
        ERR_print_errors_fp(stderr);
        SSL_free(ssl);
        close(sockfd);
        return NULL;
    }
    
    return ssl;
}

void tls_client_destroy(CustomTlsClient *client) {
    if (client->ctx) {
        SSL_CTX_free(client->ctx);
    }
    free(client);
    EVP_cleanup();
}
```

### 3. 关键算法与数据结构

#### 3.1 域名匹配算法

**当前实现**：`SortedDictionary<DomainPattern, DomainConfig>` + 线性搜索

```csharp
// DomainPattern 比较器确保更具体的模式优先
// 如 "github.com" 优先于 "*.github.com"

public bool TryGetDomainConfig(string domain, [MaybeNullWhen(false)] out DomainConfig value)
{
    value = this.domainConfigCache.GetOrAdd(domain, GetDomainConfig);
    return value != null;

    DomainConfig? GetDomainConfig(string domain)
    {
        // 线性搜索所有模式，找到第一个匹配的
        var key = this.domainConfigs.Keys.FirstOrDefault(item => item.IsMatch(domain));
        return key == null ? null : this.domainConfigs[key];
    }
}
```

**优化建议**：使用 Trie 树或后缀树加速匹配

#### 3.2 并发控制策略

| 场景 | 策略 | 位置 |
|------|------|------|
| DNS 查询 | `SemaphoreSlim` 单例 | `DnsClient.LookupAsync` |
| DNS 可用性检测 | `SemaphoreSlim` + 缓存 | `DnsClient.IsDnsAvailableAsync` |
| Handler 生命周期 | `ConcurrentDictionary` + `Lazy` | `HttpClientFactory` |
| 流量统计 | `Interlocked` + 自旋锁 | `FlowAnalyzer` |

#### 3.3 异步流处理

大量使用 `IAsyncEnumerable<T>` 进行流式处理：

```csharp
// DNS 解析结果流
public async IAsyncEnumerable<IPAddress> ResolveAsync(
    DnsEndPoint endPoint, [EnumeratorCancellation] CancellationToken cancellationToken)
{
    var hashSet = new HashSet<IPAddress>();
    await foreach (var dns in this.GetDnsServersAsync(cancellationToken))
    {
        var addresses = await this.LookupAsync(dns, endPoint, fastSort, cancellationToken);
        foreach (var address in addresses)
        {
            if (hashSet.Add(address) == true)
            {
                yield return address;  // 立即返回，不等待全部完成
            }
        }
    }
}
```
